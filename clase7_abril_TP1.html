📖 TU HOJA DE TRUCOS (AYUDA PARA HOY)

Leé este resumen antes de ir al código. Acá tenés todas las pistas que vas a necesitar para rellenar los huecos y contestar las preguntas.

  • 🔑 API Key: Es tu DNI o contraseña secreta para el servidor. Le dice a la empresa Groq quién sos para dejarte pasar.
  • ⌨️ input(): Abre una cajita para que el usuario escriba algo desde el teclado.
  • 📝 F-String (f"..."): Esa letra 'f' antes de las comillas permite meter variables adentro de un texto usando llaves { }, armando oraciones de forma automática.
  • 🛡️ Try / Except: Es nuestro escudo protector. El programa intenta conectarse. Si falla (por ejemplo, nos quedamos sin internet), ataja el error y avisa con un mensaje amigable en vez de romperse con letras rojas.
  • 🪙 Tokens: La IA no lee palabras completas, lee "pedacitos" de palabras llamados tokens. Cada vez que le preguntamos algo, gastamos tokens.

ENTRADAS DEL USUARIO ──────────────────────────────────── print("¡Bienvenida! Vamos a preparar a la IA.") personaje = input("¿Qué personaje querés que sea la IA? ") tema = input("¿Sobre qué tema querés que te hable? ") # ← COMPLETAR: ¿Qué hace la letra 'f' antes de las comillas acá? pregunta = f"Actuá como {personaje} y explicame {tema}." # ── 3. ARMADO DEL PAQUETE ────────────────────────────────────── payload = { "model": "llama-3.3-70b-versatile", "messages": [{"role": "user", "content": pregunta}], "temperature": 0.7 } # ── 4. CONEXIÓN SEGURA ───────────────────────────────────────── # ← COMPLETAR: ¿Por qué usamos try y except en esta parte? try: response = requests.post(URL, headers=headers, json=payload) data = response.json() # ← COMPLETAR: ¿Qué información estamos sacando de 'data' en estas dos líneas? respuesta = data["choices"][0]["message"]["content"] tokens = data["usage"]["total_tokens"] print(f"\n🤖 {personaje} te responde:") print(respuesta) print(f"\n(Se gastaron {tokens} tokens en total)") except Exception as e: print("⚠️ Uy, falló la conexión.
